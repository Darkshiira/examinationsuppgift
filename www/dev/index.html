Frontend 1: Examinationsuppgift

Problemställning

Hur kan man på ett visuellt sätt dela upp personer efter data som finns tillgänglig?

Jag har i min examinerande uppgift valt att bygga en hemsida där användaren kan anmäla sig(till REST APIn) till att söka efter en grupp i vår klass.

Därefter kan de söka reda på vilken grupp de hamnar i efter personlighetsfärg.

Sist kan användaren ta reda på mer information om de i klassen, eller söka efter en klasskamrat som har ett specifikt intresse.


Att kunna hämta ut olika parametrar ur data är viktigt att kunna som webbutvecklare, då flera företag använder sig av just data med olika parametrar, som ska kunna sorteras ut, användas vidare samt visualiseras på ett tydligt sätt.

T.ex. i olika webbforum, användarsidor eller varför inte dating-sidor?

Hur kan man söka i JSON-filer på en REST API-server för att plocka ut vissa specifika objekt?

Jag har här valt att använda mig av fetch för att hämta ned datan och sedan loopat igenom den för att hitta de objekt jag söker efter.

Detta eftersom data oftast inte ligger lokalt på datorn, och därför behöver hämtas hem på detta sätt.

Loopen är sedan för att gå igenom samtliga data för att hitta specifika nyckelord.

Hur kan man lägga upp data själv på en REST API-server?

Här har jag valt att återigen använda mig av fetch, genom att post:a den sträng jag skapar och konverterar den till JSON-filer genom stringify.

Detta eftersom det sällan är användbart att endast kunna använda sig av befintlig data, utan ny data måste kunna tillföras.
